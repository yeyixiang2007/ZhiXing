# 校园智能路径规划系统 - 地图化工作台重构任务安排表（M6）

> 目标：将当前“表单 + 列表”的页面形态升级为“地图驱动的专业工作台”，实现路径在地图上的可视化展示，并让管理员基于地图完成标点、连线、编辑与禁行管理。

## 1. 重构目标与范围

### 1.1 核心目标
- 以地图画布为核心交互载体，替代纯文本/表格驱动操作。
- 用户路径查询结果在地图上实时高亮，支持路线分段可视化。
- 管理员在地图上直接完成点位与道路维护（新增、修改、删除、禁行）。
- 构建专业工作台：图层、工具栏、属性面板、状态栏、操作历史与辅助网格。

### 1.2 本次范围（In Scope）
- 地图渲染引擎（2D 画布、图层、缩放、平移、选中态）。
- 用户路径展示（起终点选取、路径线段渲染、导航步骤联动高亮）。
- 管理员地图编辑（点线编辑、禁行标记、撤销重做、吸附/对齐辅助）。
- 数据同步与持久化（地图编辑后自动保存 + 备份恢复入口）。
- 性能与稳定性治理（启动、渲染、查询响应、异常兜底）。

### 1.3 暂不纳入（Out of Scope）
- GIS 三方底图接入（高德/天地图/在线瓦片）。
- 多人协同实时编辑。
- 复杂 3D 场景与室内楼层联动。

## 2. 技术路线与架构决策

### 2.1 推荐路线（本次默认采用）
- 保持 Java/Swing 技术栈，新增 `MapCanvas` 自绘层（`Graphics2D`）。
- 采用“控制器 + 视图组件”架构：
- `NavigationController`：路径计算与可视化数据输出。
- `MapController`：点线 CRUD、禁行设置、保存/恢复。
- `AuthController`：管理员登录与权限校验。
- 视图拆分：
- `MapWorkbenchView`（主工作台容器）
- `MapCanvas`（地图画布）
- `LayerPanel`（图层开关）
- `ToolPanel`（编辑工具栏）
- `PropertyPanel`（对象属性面板）
- `RoutePanel`（用户路径控制与详情）

### 2.2 数据模型扩展（增量）
- `Vertex`：补充可视化属性（图标、标签偏移、显示层级）。
- `Edge`：补充可视化属性（线型、颜色、粗细、方向箭头）。
- 新增 `MapViewState`：缩放、平移、选中对象、当前工具模式。
- 新增 `EditCommand`：支持撤销/重做（命令模式）。

## 6. 验收标准（M6）

- [ ] 用户路径必须在地图上清晰可见，包含起终点、路线高亮、分段信息。

## 7. 风险清单与应对

- 风险1：Swing 自绘复杂度高，导致渲染性能不足。
  - 应对：优先实现脏区刷新、对象缓存、文本布局缓存。
- 风险2：地图编辑与现有数据模型冲突，出现保存异常。
  - 应对：先定义编辑命令 DTO，再由 `MapController` 做统一落盘。
- 风险3：交互复杂导致误操作率高。
  - 应对：引入撤销/重做、状态提示、危险操作二次确认。
- 风险4：重构规模大，回归风险高。
  - 应对：每周固定回归窗口，保持 `qa.ps1` 全量通过作为门禁。
